XPressFreedom Embedded

This is a project in construction yet; After the development of the SMS-Access system, some changes in the XpressFreedom board can make the whole scheme of security gates being totally controlled by just one system, without the need of a central panel to open the Gates after it passes through the board. I am in charge to make this project work, and to make this happen I am using C programming for Embedded systems.

I am using the idea of saving card data in a external EEPROM memory linked to the XPressFreedom board, so the client (possibly an web one) will send the card data to save in the board, and will send access-requests from the users trying to get into a specific gate.

This project may look simple, but you have to consider a few variables that came from the limitations of the embedded system:

  • There is not so much RAM memory to save the cards data all in just one interaction with the client software.
  • There's not enough ROM memory to increase substantially the code.
  • the EEPROM memory is not supposed to be written a lot of times
  • the reading, searching and writing from the EEPROM must be as fast as possible

I will have to send the card data chunk by chunk because of the memory problem, and I will have to use a fast search method (binary search) that includes a ordering method that is also supposed to be fast. And not everything will be programmed in the board, so I have to change the SDK to work with every client we intent to make (SMS-Access is one, an standalone security system would be another, etc).
